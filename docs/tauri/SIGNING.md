# Tauri Update Signing

## Why Signing Is Required

Every update binary must be signed with your private key before shipping. The app verifies the signature against the public key baked into the build (`tauri.conf.json`). Without a valid signature, Tauri rejects the update — this prevents tampered or malicious updates reaching users.

## One-Time Setup

### Step 1 — Generate the key pair

Run once on your machine:
```bash
bunx tauri signer generate -w ~/.tauri/manga-dl.key
```

Output:
```
✔ Generating keypair
Your keypair was generated successfully
Private: ~/.tauri/manga-dl.key (keep this secret)
Public key: dW50cnVzdGVkIGNvbW1lbnQ6IG1pbmlzaWduIHB1YmxpYyBrZXkgQ...
```

### Step 2 — Add public key to tauri.conf.json

In `frontend/src-tauri/tauri.conf.json`:
```json
"plugins": {
  "updater": {
    "pubkey": "dW50cnVzdGVkIGNvbW1lbnQ6IG1pbmlzaWduIHB1YmxpYyBrZXkgQ...",
    "endpoints": [
      "https://github.com/zenmisan/manga-dl/releases/latest/download/latest.json"
    ]
  }
}
```

### Step 3 — Add secrets to GitHub Actions

Go to: GitHub repo → Settings → Secrets and variables → Actions → New repository secret

| Secret name | Value |
|---|---|
| `TAURI_PRIVATE_KEY` | Contents of `~/.tauri/manga-dl.key` |
| `TAURI_KEY_PASSWORD` | Password you set during key generation (leave blank if none) |

### Step 4 — Back up your private key

Store `~/.tauri/manga-dl.key` somewhere safe (password manager, encrypted drive). If lost, you cannot sign future updates — all existing installs will stop receiving updates.

## GitHub Actions Workflow

Add to `.github/workflows/release.yml`:

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  release:
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
    runs-on: ${{ matrix.os }}

    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v2

      - name: Install Rust
        uses: dtolnay/rust-toolchain@stable

      - name: Install dependencies (Linux)
        if: matrix.os == 'ubuntu-latest'
        run: |
          sudo apt-get update
          sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf

      - name: Install frontend deps
        working-directory: frontend
        run: bun install

      - name: Build and release
        uses: tauri-apps/tauri-action@v0
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          TAURI_PRIVATE_KEY: ${{ secrets.TAURI_PRIVATE_KEY }}
          TAURI_KEY_PASSWORD: ${{ secrets.TAURI_KEY_PASSWORD }}
        with:
          projectPath: frontend
          tagName: ${{ github.ref_name }}
          releaseName: manga-dl ${{ github.ref_name }}
          releaseBody: |
            See the changelog for details.
          releaseDraft: false
          prerelease: false
```

This workflow:
- Triggers on any `v*` tag push (e.g. `git tag v1.0.1 && git push --tags`)
- Builds for Windows, macOS, and Linux in parallel
- Signs all binaries with your private key
- Creates a GitHub Release with all assets + `latest.json` manifest
- The Tauri updater in shipped apps then finds this release automatically

## Releasing a New Version

```bash
# 1. Bump version in frontend/package.json AND frontend/src-tauri/tauri.conf.json
#    (both must match)

# 2. Push a version tag
git tag v1.0.1
git push origin v1.0.1

# 3. GitHub Actions builds, signs, and publishes the release automatically
```

## Key Concepts

| Term | Meaning |
|---|---|
| Private key | Signs your release binaries — keep secret, never commit |
| Public key | Verifies signatures — safe to be public, baked into app |
| `latest.json` | Update manifest generated by `tauri-action` — lists versions, URLs, signatures per platform |
| Signature (`.sig`) | Per-binary file generated during signing — uploaded alongside each binary |
